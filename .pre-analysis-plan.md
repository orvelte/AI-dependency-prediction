# Pre-Analysis Plan

**Date written:** 2026-04-29  
**Updated with data-acquisition context:** 2026-04-30  
**Status:** LOCKED — do not modify after modelling begins

---

## Research Question

Can linguistic features extracted from early user sessions (sessions 1–3) predict dependency-adjacent interaction patterns in late sessions (session 10+) in longitudinal human-AI chat logs?

---

## Dataset

- **Source:** WildChat-1M (`allenai/WildChat-1M` via Hugging Face)
- **Acquisition method:** Hugging Face streaming API, filtered to qualifying users only (full 7 GB raw dataset was NOT downloaded locally)
- **Filtered file location:** `data/interim/wildchat_longitudinal.parquet`
- **Timestamp span:** 2023-04-09 to 2024-04-29 (386 days)
- **Redaction rate:** 0.9% of conversations (top-level `redacted == True`)
- **Observed refusal rate (keyword lower bound):** 3.2% of conversations
- **Turn count distribution:** median 2, mean 4.5, max 498 (heavily right-skewed)

---

## Cohort Definition

- **Inclusion criterion:** Users with ≥ 10 conversations (`hashed_ip` grouping)
- **Final cohort:** 12,577 unique users, 509,861 conversations
- **Average conversations per qualifying user:** ~40.5
- **Rationale for ≥ 10 threshold:** provides non-overlapping early (sessions 1–3) and late (sessions 10+) windows; drop-off curve showed a natural inflection between 5 and 10; 12,577 users is sufficient for 10-fold cross-validation

---

## Session Construction

- **Unit of analysis:** user × conversation (conversations treated as sessions)
- **User grouping field:** top-level `hashed_ip` (most common IP per conversation)
- **Conversation ID:** derived from `conversation[0]["turn_identifier"]` — the dataset has no native unique conversation-level ID (`conversation_hash` is not unique per the dataset card)
- **Session number:** chronological rank of conversation within a user's history (1 = earliest), assigned by sorting on top-level `timestamp` (UTC, timestamp of last turn in conversation)

---

## Pre-Specified Exclusions

- Conversations where top-level `redacted == True` are EXCLUDED as the first filter step
- **Rationale:** redacted content (0.9%) may have distorted linguistic markers due to PII anonymisation; exclusion keeps feature extraction clean
- This exclusion is applied in `filter.py` before any feature extraction

---

## Predictor and Outcome Windows

- **Early-session features (predictors):** sessions 1–3 (first 3 conversations chronologically per user)
- **Late-session features (outcome):** sessions 10+ (conversations 10 onwards chronologically per user)

---

## Hypotheses (Directional, Pre-Specified)

Effect directions are theoretically motivated by the goal-directed → habitual transition model.

| ID | Hypothesis | Expected direction |
|----|-----------|-------------------|
| H1 | First-person plural pronoun rate in sessions 1–3 predicts high-dependency classification in sessions 10+ | Positive |
| H2 | Self-disclosure marker rate in sessions 1–3 predicts high-dependency classification | Positive |
| H3 | Refusal resistance rate in sessions 1–3 predicts high-dependency classification | Positive |
| H4 | Question-to-statement ratio in sessions 1–3 predicts high-dependency classification | Positive (more questions = more help-seeking) |
| H5 | Mean session length slope from session 1→3 predicts high-dependency classification | Positive (escalating engagement) |
| H6 | Vocabulary richness (TTR/MTLD) in sessions 1–3 is *negatively* associated with high-dependency | Negative (dependency → narrowing discourse) |

---

## Dependency Outcome Variables (do not add post-hoc)

| Variable | Operationalisation |
|----------|-------------------|
| Session length | Number of user turns per conversation |
| Return interval | Time between conversation end and next conversation start (requires timestamp sorting per user) |
| Linguistic intimacy | First-person plural rate, self-disclosure markers, relational language |
| Refusal resistance | Rate of user rephrasing after an AI refusal (see caveats below) |
| Engagement escalation | Slope of session length over time |

---

## Outcome Variable Definition (Pre-Specified)

**"High-dependency" label** applied to a user if they meet ≥ 2 of the following criteria in sessions 10+:

1. Return interval < median of cohort (i.e., returns faster than average)
2. Session length > 75th percentile of cohort
3. Refusal resistance rate > 0 (at least one rephrase-after-refusal detected)
4. First-person plural rate > cohort mean in late sessions

This composite rule is fixed. Threshold sensitivity is explored in Phase 5 (robustness) but the primary label uses this rule.

---

## Refusal Resistance Caveats (Pre-Specified)

- Refusal detection uses keyword matching — this is a **lower bound estimate**; true refusal rate is likely higher than the observed 3.2%
- Refusal resistance is only computed for users who encountered **≥ 3 refusals** across their conversation history (minimum exposure threshold for a reliable score)
- Users below this threshold have refusal resistance treated as **missing/NA, not zero**
- Refusal resistance is a **secondary feature** given data sparsity; primary features are intimacy markers, session length, and return interval
- First-pass implementation: crude proxy (did user send any follow-up after a refusal?)
- Refinement if time allows: semantic similarity between pre- and post-refusal messages to confirm same underlying intent

---

## Pre-Specified Feature Transforms

- **Session length:** expected to be heavily right-skewed (median turns = 2, mean = 4.5, max = 498) — apply **log transform** before modelling
- **Return interval:** likely right-skewed — apply **log transform** before modelling
- Any additional transforms applied must be documented here with reasoning before modelling begins

## Analysis Plan

### Phase 1 — Baselines
- Fit logistic regression for each feature in H1–H6 individually
- Report AUC, precision, recall, Cohen's d for each
- Apply Benjamini-Hochberg FDR correction across all feature-level tests

### Phase 2 — Main Model
- Fit L1-regularised logistic regression on full feature set from sessions 1–3
- Cross-validate at user level, K=10 folds (users must not appear in both train and test)
- Primary metric: AUC-ROC
- Secondary metrics: balanced accuracy, precision, recall

### Phase 3 — Robustness
- **Permutation test:** shuffle dependency labels, refit 1000×, compare observed AUC to null distribution
- **Negative control:** include punctuation rate as a theoretically unmotivated feature; confirm non-significance
- **Sensitivity — label threshold:** vary composite label threshold (1–4 criteria) and check rank stability of top features
- **Sensitivity — session threshold:** vary minimum session count (≥ 8, ≥ 10, ≥ 12) and confirm results are stable

---

## What Would Invalidate the Study

- Fewer than 200 users in the longitudinal cohort — insufficient power
- Extreme class imbalance (>90:10) without successful stratification
- Permutation test AUC falling within the 95th percentile of the null distribution
- Negative control feature ranking in top 3 by feature importance
